# Ecommerce Sales Data Analysis

## Project Overview

This project analyzes an **Ecommerce Sales Dataset** to uncover valuable business insights using SQL.
The goal of the project is to demonstrate **data validation, data cleaning, exploratory analysis, and business insights generation** using real-world transactional data.

The dataset was sourced from **Kaggle** and contains **100,000 ecommerce transactions**, including product information, customer demographics, payment methods, and sales data.

This project showcases **SQL skills from basic to advanced level**, including aggregations, window functions, ranking, customer segmentation, and time-series analysis.

---

## Dataset

Dataset Name: **ecommerce_sales_data**

## Key Business Insights

### 1. Total Revenue

The dataset generated approximately **$106.66 million** in total revenue after discounts.

### 2. Top Performing Product Category

**Books** generated the highest overall revenue among all product categories.

### 3. Payment Method Analysis

**Bank Transfer** was the most frequently used payment method among customers.

### 4. Monthly Revenue Trends

Revenue remained relatively stable between **$8M–$9M per month**, indicating consistent sales performance.

### 5. High Value Customers

Top customers spent around **$30K**, indicating strong customer purchasing power.

### 6. Average Order Value

The **Average Order Value (AOV)** was approximately **$1066 per transaction**.

### 7. Regional Performance

Certain locations such as **New Michael** and **Lake Michael** generated the highest revenue.
